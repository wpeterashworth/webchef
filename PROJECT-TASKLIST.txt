PROJECT SUMMARY
---------------
A Duolingo-style cooking education app. Users sign in, take short lessons
(ingredient substitutions, safe cooking temperatures, prep/cook time
estimation), get instant feedback, earn scores/XP, build streaks, and level
up through tiers with themed names.

TECH STACK (already decided - see the ADRs)
  Frontend:  Svelte / SvelteKit
  Backend:   Node.js + Express  OR  SvelteKit API routes  (DECISION PENDING)
  Database:  PostgreSQL on Supabase (shared team instance)
  Auth:      Supabase Auth
  Hosting:   Render

 PHASE 4 - BACKEND API   (build endpoints in priority order)
==============================================================================
  Build P1 endpoints first (lessons + attempts), then P2 (progress/streaks),
  then P3 (challenges), then admin/social last.

  P1 - Core learning loop:
4.1  GET  /api/lessons                       list available lessons
4.2  GET  /api/lessons/:lessonId             lesson details + questions
4.3  POST /api/lessons/:lessonId/attempts    submit answers, score + feedback
       -> This endpoint also updates score, XP, streak, and level.

  P2 - Progress & streaks:
4.4  GET  /api/progress                       progress summary for dashboard
4.5  GET  /api/progress/:lessonId             progress for one lesson
4.6  GET  /api/streaks                         current streak + history
4.7  Level-up logic: when score crosses a threshold, level +1 and update the
     displayed level name (FR-011 / SC-006).

  P3 - Challenges:
4.8  POST /api/challenges/:challengeId/attempts   submit a challenge result
4.9  GET  /api/recommendations                next lessons / weak-area review

  Later / additional:
4.10 POST /api/admin/lessons  +  PUT /api/admin/lessons/:lessonId  (authoring)
4.11 POST /api/shares/progress       shareable progress snapshot
4.12 POST /api/challenges/invite     challenge a friend
